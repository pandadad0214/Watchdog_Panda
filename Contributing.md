## 1. Reporting Issues  
If you find a bug, inconsistency, or improvement opportunity:

1. Go to the **Issues** tab.
2. Choose the appropriate template (Bug, Feature Request, Enhancement).
3. Provide:
   - Clear description  
   - Steps to reproduce (if applicable)  
   - Expected vs actual behavior  
   - Logs or screenshots (optional)  

Be concise but thorough ‚Äî Elder Watchdog appreciates clarity.

---

## 2. Submitting Pull Requests (PRs)

### Before you open a PR:
- Ensure your branch is up to date with `main`.
- Follow the commit message style below.
- Make sure your changes are scoped and intentional.
- Run any tests or linting tools (if applicable).

### PR Requirements:
- A clear title describing the change.
- A short summary of what the PR does.
- Reference any related issues (e.g., `Closes #12`).
- No unrelated changes bundled together.

### PR Review Process:
- Maintainers will review for clarity, stability, and alignment with Federation philosophy.
- Requested changes should be addressed promptly.
- Once approved, maintainers will merge the PR.
